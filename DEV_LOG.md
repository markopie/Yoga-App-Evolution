## [2026-05-09] - Session [07]
**Goal:** Document future UI cleanup ‚ÄĒ curriculum day vs composed part vs source sequence label distinction.

**Future UI Cleanup Note ‚ÄĒ Curriculum Display Labels**

A curriculum day may reuse source sequences whose original titles are context-specific to their source book's chapter structure (e.g. "Week 1 Day 2", "Week 1 and 2"). If these source titles are surfaced as the curriculum day label, the UI becomes confusing because the user sees a source-chapter title instead of a curriculum-path position.

Three label types must be kept distinct in any curriculum-facing UI:

- **Curriculum day label** ‚ÄĒ derived from the user's position in `draft_v1`, e.g. "Week 5 Day 3 ‚ÄĒ Foundation Practice". This is the user-facing identity of the day and must never be replaced by a source title.
- **Part label** ‚ÄĒ the role of each composition part within a composed day, e.g. "Part 1: Foundation Asana" / "Part 2: Short Pranayama". Drawn from the `practice_composition` role field via `roleLabel()` (see `prototypes/curriculum-roadmap/roadmap.js`).
- **Source label** ‚ÄĒ the original source-sequence title, shown only in secondary/detail context, e.g. "Source: How to Use Yoga ‚ÄĒ Week 1 Day 2". Drawn from `program_curriculum.source_name` and `source_reference`.

**Design pattern from the roadmap prototype that works well here:**

The prototype's `roleLabel()` + `nodeTypeLabel()` separation (in `roadmap.js`) already provides the right vocabulary. In the production curriculum UI, the same three-tier hierarchy should be applied:

```
[Curriculum position] Week 5 ¬∑ Day 3             ‚Üź always curriculum-derived
[Node type badge]     Combined Practice            ‚Üź from nodeTypeLabel()
  Part 1: Asana       Light on Yoga ‚ÄĒ Week 5      ‚Üź roleLabel() + source_reference
  Part 2: Short Pranayama  Light on Pranayama ‚ÄĒ Week 3
                      Source: How to Use Yoga ‚ÄĒ Week 1 Day 2   ‚Üź tertiary, detail only
```

The source title belongs in the detail/secondary layer only. It should never be promoted to become the curriculum day heading.

**Files to update when implementing:**
- `src/ui/curriculumUI.js` ‚ÄĒ curriculum day title construction
- `prototypes/curriculum-roadmap/roadmap.js` ‚ÄĒ already has the correct label helpers; reuse in production
- Any rating overlay or completion summary that shows the practice title

**Theme/style notes:**
- Use `--color-text-primary` for the curriculum day label (strong hierarchy)
- Use `--color-text-secondary` for part labels
